 3.

在CoppeliaSim中，可以使用Lua或Python腳本來創建基本類型（Primitive Type）的對象，並通過編程的方式來控制它們的位置。  以下是在CoppeliaSim中使用Lua腳本創建Primitive Type對象並設置位置的示例：

-- 创建一个立方体Primitive Type对象
cubeHandle=sim.createPureShape(1,16,{0.1,0.1,0.1},0.2)

-- 将对象位置设置为x=1、y=2、z=3的位置
sim.setObjectPosition(cubeHandle,-1,{1,2,3})

上述代碼中，第一行創建了一個大小為0.1×0.1×0.1的立方體對象，並將其存儲在cubeHandle變量中。第二行使用sim.setObjectPosition函數將該對象的位置設置為x=1、y=2、z=3的位置。  以下是在CoppeliaSim中使用Python腳本創建Primitive Type對象並設置位置的示例：


导入Python API库
import sim

# 连接到CoppeliaSim
sim.simxFinish(-1)
clientID=sim.simxStart('127.0.0.1',19999,True,True,5000,5)

# 创建一个立方体Primitive Type对象
errorCode, cubeHandle = sim.simxCreatePureShape(clientID,1,16,[0.1,0.1,0.1],0.2, None, sim.simx_opmode_blocking)

# 将对象位置设置为x=1、y=2、z=3的位置
errorCode = sim.simxSetObjectPosition(clientID, cubeHandle, -1, [1,2,3], sim.simx_opmode_blocking)

上述代碼中，首先通過simxStart函數連接到CoppeliaSim，並使用simxCreatePureShape函數創建一個大小為0.1×0.1×0.1的立方體對象。然後使用simxSetObjectPosition函數將該對象的位置設置為x=1、y=2、z=3的位置。  無論使用Lua還是Python，都可以通過類似的方式來創建Primitive Type對象並設置位置。

4.

CoppeliaSim中，.ttt文件和.simscene.xml文件都是場景文件的格式，但它們在文件結構和用途上有一些不同。  .ttt文件是CoppeliaSim的早期版本使用的格式，它是一個二進製文件，包含場景中所有對象的完整信息，包括位置、大小、材質、關節等。這種格式的優點是加載速度快，但缺點是不利於版本控制和與其他軟件集成。  相比之下，.simscene.xml文件是CoppeliaSim較新版本使用的格式，它是一個基於XML的文本文件，包含了場景的所有信息，可以在文本編輯器中輕鬆編輯和查看。這種格式的優點是易於版本控制和與其他軟件集成，但缺點是加載速度相對較慢。  因此，通常情況下建議使用.simscene.xml格式的場景文件，因為它更加靈活，可維護性更高，而且能夠與其他軟件集成。如果您正在使用較早的CoppeliaSim版本，可能需要使用.ttt格式的場景文件，但建議儘早升級到最新版本以獲得更好的性能和更多的功能。
